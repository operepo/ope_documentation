# App Descriptions
- OPE LMS App - App to sync with canvas and show offline course materials as well as lock down security on the laptop, includes LMS App, credential app, security service
- Git - source control and file tracking, used to update lms app and ensure files haven't changed
- Windows 10 - Pro or enterprise required, Home NOT supported
- Office 2016 - Word, Excel, Powerpoint ok, Access not approved, VBA/developer not approved
- Winrar - App to zip/unzip archives, understands various archive formats
- Flash Player - Adobe flash player
- Adobe Reader - For reading PDF files
- Java SE Runtime
- VLC Media Player - Video playing software the supports multiple formats and updated codecs
- Firefox - Needed for displaying Canvas content through LMS app (offline PDF, web pages, videos, etc...)
- Gimp - Graphic Design - open source, similar to photoshop
- Blender - 3D modeling - open source, similar to Maya - 3d landscapes, texturemapping and animation tools
- Inkscape - Graphic Design Vector Graphics - open source, similar to illustrator
- QuickBooks - Accounting software
- Chrome Browser
- Open SCAD - Cad program - open source, similar to autocad
- Notepad++ - text editor, understand code languages (shows with color) but requires seperate compiler/development tools to run code

# For MCC
Chrome - They would be learning to make web pages and get HTML/CSS to layout properly, chrome is one of the best browsers to learn/debug pages on
·         Node.JS - This is the server side javascript environment - lets them use javascript for server side as well as the client (web browser). Should be able to limit the packages/libraries needed to just the assignment (similar to what we did with python - no win api libs)
·         Postman - A tool that lets you bounce off a web server (like node) and see the protocol that is running, SUPER helpful when building web apps. Again - limited where it can go by firewall/etc....
·         Media Player (Windows’ or VLC?)  - VLC is on the approved list already
·        on hold —  Sublime Text Editor - Another text editor like notepad++, still need compilers or other tools to actually run code.
·         on hold —  check w last mile — Visual Studio - Following up to find out which packages they want/need, we want to limit driver development kits for sure, if they are just installing the basic nodejs support it is less of an issue. Like python - we would want to limit things that give more access to the system.
·         Notepad++ - Already on the approved list
·         SQLite - Local database that runs as part of the app - don't need a separate database "server". It is just a library you add to your app (e.g. nodejs package). They WILL want an SQLite app that allows them to open the file and view it.
·         MongoDB - Key value document database (put "mydoc" -> "sdlfjdlkfjlds").  Very common. Used to quickly put and get values.  MongoDB itself doesn't run code and is pretty simple. Will work as localhost, but network limits keep it being insecure.
